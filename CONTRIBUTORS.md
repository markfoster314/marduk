## How to Contribute

Thank you for your interest in contributing to Marduk! We welcome contributions from the community.

### Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/marduk.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Run tests: `npm test`
6. Run linting: `npm run lint`
7. Build: `npm run build`
8. Commit: `git commit -m "feat: your feature description"`
9. Push: `git push origin feature/your-feature-name`
10. Open a Pull Request

### Development Setup

npm install
npm run storybook # View components
npm test # Run tests
npm run lint # Check code quality### Contribution Guidelines

- Follow the existing code style
- Write tests for new features
- Update documentation as needed
- Use conventional commits (feat:, fix:, docs:, etc.)
- Ensure all tests pass before submitting PR
- Keep PRs focused on a single feature/fix

### Pull Request Process

1. Ensure your code passes all tests and linting
2. Update the README.md if needed
3. Update CHANGELOG.md following Keep a Changelog format
4. The PR will be reviewed by the maintainer
5. Once approved, it will be merged into main
